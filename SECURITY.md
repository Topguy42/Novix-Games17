# Security Policy

## Supported Versions
We actively maintain the latest branch of PeteZahGames.  
Other branches may be experimental. Always deploy the stable version.

|  Branch | Supported |
|---------|-----------|
| main    | ✅        |
| others  | ❌        |

---

## Reporting a Vulnerability
If you discover a security vulnerability in this project:

1. **Do not open a public issue.**
2. Report privately using one of these methods:
   - GitHub [Security Advisories](https://docs.github.com/en/code-security/security-advisories/repository-security-advisories).
   - Email: [contact@petezahgames.com](mailto:contact@petezahgames.com?subject=Security%20Advisory&body=%28Put%20title%20of%20advisory%20here%29%0ACVE%20identifier%3A%20%28Put%20it%20here%2C%20if%20there%20is%20none%20leave%20it%20blank%29%0A%0ADescription%3A%0A%28Fill%20this%20in%29%0AWrite%0APreview%0AImpact%0A%28What%20kind%20of%20vulnerability%20is%20it%3F%20Who%20is%20impacted%3F%29%0A%0APatches%0A%28Has%20the%20problem%20been%20patched%3F%20What%20versions%20should%20users%20upgrade%20to%3F%29%0A%0AWorkarounds%0A%28Is%20there%20a%20way%20for%20users%20to%20fix%20or%20remediate%20the%20vulnerability%20without%20upgrading%3F%29%0A%0AReferences%0A%28Are%20there%20any%20links%20users%20can%20visit%20to%20find%20out%20more%3F%29%0A%0AAffected%20products%0A%0ASelect%20an%20ecosystem%0APackage%20name%0Ae.g.%20example.js%0AAffected%20versions%0Ae.g.%20%3C%201.2.3%0APatched%20versions%0Ae.g.%201.2.3%0A%0ASeverity%0A%28Put%20what%20you%20believe%20the%20severity%20is%20in%20the%20following%20format%3A%0ACVSS%3A3.1%2FAV%3A_%2FAC%3A_%2FPR%3A_%2FUI%3A_%2FS%3A_%2FC%3A_%2FI%3A_%2FA%3A_%29%0A%0AWeaknesses%0A%28Put%20weaknesses%20%20here%29%0ACredits%0A%28Add%20people%20who%20contributed%29).

Please include:
- A clear description of the vulnerability.
- Steps to reproduce.
- Potential impact.
- Suggested mitigation if available.

We will acknowledge reports within 72 hours and aim to provide a fix within 30 days.

---

## Disclosure Policy
- Vulnerabilities will be disclosed publicly only after a patch is released.
- Critical issues may be patched immediately with a hotfix.
- Minor issues will be scheduled for the next release cycle.
